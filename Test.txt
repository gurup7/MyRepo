# Multi-Repository Maven & JDK 21 Migration Assessment

Analyze all repositories in the current workspace and identify every `pom.xml`.

For each repository:

## Assessment Scope

1. Extract major direct dependencies (groupId:artifactId, version, scope).
2. Check known vulnerabilities (CVEs) and include:
   - Severity (Critical/High/Medium/Low)
   - Fixed version
3. Assess JDK 21 compatibility for:
   - Dependencies
   - Maven plugins
4. Review JDK 21 readiness of key plugins:
   - maven-compiler-plugin
   - maven-surefire-plugin
   - maven-failsafe-plugin
   - jacoco-maven-plugin
   - spring-boot-maven-plugin
   - Other significant plugins found
5. If Spring Boot is used:
   - Identify current version
   - Recommend JDK 21 compatible version
   - Highlight required Spring Framework upgrades
6. Flag potential breaking changes or code remediation requirements.

---

## Output Format

### 1. Executive Summary (One Row Per Repository)

| Repo | Risk (High/Medium/Low) | JDK 21 Readiness | Critical Findings | Recommended Action |
|------|------------------------|------------------|-------------------|-------------------|

---

### 2. Detailed Findings

| Repo | Dependency / Plugin | Current Version | Vulnerability (CVE) | Severity | JDK 21 Compatible (Y/N) | Recommended Version | Notes |
|------|---------------------|-----------------|---------------------|----------|-------------------------|---------------------|-------|

Only include:
- Major dependencies
- Vulnerable dependencies
- Dependencies/plugins requiring upgrades for JDK 21

---

### 3. JDK 21 Upgrade Insights

| Repo | Observation | Impact | Recommendation |
|------|-------------|---------|----------------|
| Repo Name | Key migration blockers, outdated libraries, plugin issues, Spring Boot concerns, security risks | High/Medium/Low | Recommended upgrade path |

---

## Rules

- Group all findings by repository name.
- Prioritize security vulnerabilities and JDK 21 blockers.
- Recommend the safest target version rather than the latest version.
- Clearly identify dependencies requiring code changes versus simple version upgrades.
- Keep the report concise and actionable.
