# Snyk Vulnerability Remediation Skill

## Purpose

This file instructs GitHub Copilot Chat on how to remediate Snyk-reported
dependency vulnerabilities in Citi CHUB Spring Boot microservices. When you
ask Copilot to fix vulnerabilities, it reads this file and applies the
correct fix strategy.

## Scope

- Maven pom.xml dependency version upgrades
- JDK 17 to 21 migration (properties + compiler plugin)
- Does NOT modify Java source code
- Does NOT add or remove dependencies

## How to Invoke

Open this repo in VS Code and use any of these prompts in Copilot Chat:

```
@workspace Fix Snyk vulnerabilities
@workspace Upgrade dependencies to secure versions
@workspace Apply CVE remediation
@workspace Migrate to JDK 21
```

---

## Project Structure (Citi CHUB Microservices)

These microservices use a **custom Citi parent BOM**, NOT the standard
Spring Boot parent. This is critical for the fix strategy.

```xml
<parent>
    <groupId>com.citi.169073.ccp.rel</groupId>
    <artifactId>ccp-spring-boot-starter-parent-master</artifactId>
    <version>2026.6.13.RELEASE</version>
</parent>
```

### Key POM Characteristics:
- Parent BOM: `ccp-spring-boot-starter-parent-master` (Citi's internal BOM)
- Packaging: WAR (deployed to external Tomcat)
- Most Spring/Boot dependency versions are inherited from the parent BOM
- Some dependencies have explicit versions in the POM
- JDK version is set in `<properties>` AND in `maven-compiler-plugin` config
- Tomcat is excluded from spring-boot-starter-web, then re-added as provided

## Fix Strategy (Priority Order)

### Priority 1: Upgrade the Parent BOM Version

The Citi parent BOM (`ccp-spring-boot-starter-parent-master`) manages versions
for Spring Framework, Spring Boot, Micrometer, Netty, Tomcat, and other core
dependencies. Upgrading this ONE version fixes the majority of Snyk CVEs.

```xml
<parent>
    <groupId>com.citi.169073.ccp.rel</groupId>
    <artifactId>ccp-spring-boot-starter-parent-master</artifactId>
    <version>LATEST_AVAILABLE_VERSION</version>  <!-- Check Citi Nexus for latest -->
</parent>
```

**Action:** Check Citi's internal Nexus/Artifactory for the latest version of
`ccp-spring-boot-starter-parent-master` that includes patched Spring 6.2.19+
or Spring 7.0.8+ dependencies.

**If no newer parent BOM is available**, proceed to Priority 2.

### Priority 2: Add Version Properties to Override Parent BOM

If the parent BOM hasn't been updated yet, you can force specific versions
by adding properties that the parent BOM recognizes:

```xml
<properties>
    <!-- Existing properties... -->
    
    <!-- ADD THESE to override parent BOM versions -->
    <spring-framework.version>6.2.19</spring-framework.version>
    <spring-retry.version>2.0.13</spring-retry.version>
    <micrometer.version>1.15.12</micrometer.version>
    <netty.version>4.1.135.Final</netty.version>
    <tomcat.version>10.1.55</tomcat.version>
</properties>
```

NOTE: This only works if the parent BOM is configured to respect these property
names. Verify by running `mvn dependency:tree` after adding them. If versions
don't change, use Priority 3 instead.

### Priority 4: Upgrade JDK 17 to 21

JDK version is set in THREE places in these POMs. All three must be updated:

**Place 1 — Properties block:**
```xml
<properties>
    <java.version>21</java.version>
    <maven.compiler.target>21</maven.compiler.target>
    <maven.compiler.source>21</maven.compiler.source>
</properties>
```

**Place 2 — maven-compiler-plugin configuration:**
```xml
<plugin>
    <groupId>org.apache.maven.plugins</groupId>
    <artifactId>maven-compiler-plugin</artifactId>
    <version>3.15.0</version>
    <configuration>
        <source>21</source>
        <target>21</target>
    </configuration>
</plugin>
```

**Place 3 — Dockerfile base image:**
```dockerfile
# Change from jdk17 to jdk21
FROM tomcat:10.1-jdk21-temurin-jammy
```

## Validation After Fix

1. Run: `mvn clean compile -DskipTests`
2. Run: `mvn dependency:tree | findstr "spring-core"` — verify version is 6.2.19+
3. Run: `mvn dependency:tree | findstr "micrometer-core"` — verify version is 1.15.12+
4. Run: `mvn dependency:tree | findstr "netty-handler"` — verify version is 4.1.135+
5. If build passes, commit and push

---

## Maintenance

When new Snyk findings appear:
1. First check if a newer `ccp-spring-boot-starter-parent-master` is available
2. If yes, update parent version — this is the cleanest fix
3. If no, add/update the `<dependencyManagement>` overrides above
4. Update this skill file with new target versions
5. Re-run the automation script or Copilot Chat
